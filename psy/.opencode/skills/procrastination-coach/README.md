# Procrastination Coach Skill - Documentation

## Что это за навык?

**Procrastination Coach** — это специализированный навык для Claude, который превращает его в опытного психолога-коуча по работе с прокрастинацией и перфекционизмом.

## Основные возможности

### 1. Психологический анализ
- Выявление психологических паттернов (перфекционизм, страх неудачи, синдром самозванца и др.)
- Определение глубинных убеждений, блокирующих действия
- Анализ эмоциональных триггеров прокрастинации

### 2. Когнитивная работа
- Переформулирование деструктивных убеждений
- Техники работы с катастрофизацией
- Развитие здорового отношения к ошибкам и несовершенству

### 3. Практические стратегии
- 30+ проверенных техник борьбы с прокрастинацией
- Индивидуальный подбор методов под конкретные паттерны
- Конкретные инструменты для преодоления барьеров

### 4. Планирование действий
- Детальная декомпозиция задач
- Создание реалистичного плана с микрошагами
- Система поддержки и профилактика срывов

### 5. 🆕 Логирование сессий для исследований
**Новая функция:** Автоматическое создание структурированных логов сессий.

## Функционал логирования

### Зачем это нужно?

Логирование позволяет:
- **Отслеживать прогресс** — вернуться к инсайтам и стратегиям
- **Сохранять план** — иметь справочник техник и шагов
- **Проводить исследования** — анализировать паттерны прокрастинации
- **Улучшать подход** — понимать, какие методы работают лучше
- **Делиться данными** — анонимно способствовать исследованиям

### Типы логов

#### 1. Полный лог сессии (Detailed Session Log)
Комплексный документ, включающий:
- Начальную ситуацию и состояние пользователя
- Психологический анализ (паттерны, убеждения, триггеры)
- Работу с когнитивным рефреймингом
- Выбранные техники и стратегии
- Детальный план действий
- Оценку результатов сессии
- **Исследовательские заметки** — наблюдения о паттернах, эффективности техник

**Когда использовать:** После полной коучинговой сессии, для исследований, для детального отслеживания.

**Пример:** См. `session_log_example.md` в навыке

#### 2. Быстрый лог (Quick Log)
Краткая заметка на 1 страницу:
- Задача
- Главная проблема
- Выявленный паттерн
- Ключевой инсайт
- Первое действие

**Когда использовать:** Для коротких консультаций, быстрых проверок, когда времени мало.

### Как использовать логирование

#### Базовое использование:

1. **Пройдите коучинговую сессию** с навыком
2. **После завершения** Claude предложит создать лог
3. **Согласитесь** — будет создан markdown-файл
4. **Сохраните файл** для личного использования

#### Для исследований:

1. **Соберите несколько логов** (10-20+ сессий)
2. **Изучите** `research_guide.md` — руководство по анализу
3. **Анализируйте паттерны**:
   - Какие комбинации паттернов встречаются чаще?
   - Какие техники наиболее эффективны?
   - Какие инсайты типичны для определённых профилей?
4. **Агрегируйте данные** — используйте таблицы и метрики
5. **Улучшайте навык** — обновляйте техники на основе находок

### Конфиденциальность и этика

#### Автоматическая анонимизация

При создании логов для исследований:
- ✅ Заменяйте имена на псевдонимы
- ✅ Удаляйте названия компаний
- ✅ Обобщайте специфичные детали проектов
- ✅ Сохраняйте достаточно контекста для анализа

#### Согласие пользователя

- Всегда информируйте о возможном использовании для исследований
- Получайте явное согласие перед использованием данных
- Позволяйте удалить данные по запросу
- Делитесь находками с участниками, если интересно

#### Безопасное хранение

- Логи содержат чувствительную психологическую информацию
- Храните в безопасном месте
- Ограничьте доступ
- Имейте политику удаления данных

Просто опишите проблему, например:

### 3. Следуйте процессу
Claude проведёт вас через:
- Исследование ситуации (вопросы)
- Анализ паттернов

- Создание плана

После завершения:
- Claude предложит создать лог сессии

- Получите markdown-файл для сохранения

## Исследовательское использование

### Сценарии применения

**1. Индивидуальное исследование:**
- Анализируйте собственные паттерны прокрастинации
- Отслеживайте, что работает для вас лично
- Развивайте самопознание

**2. Групповое исследование:**
- Собирайте логи от группы (с согласия!)
- Анализируйте общие паттерны
- Выявляйте эффективные техники для разных типов

**3. Академическое исследование:**
- Используйте для психологических исследований
- Анализируйте когнитивные паттерны
- Публикуйте находки (с анонимизацией!)

**4. Улучшение практики:**
- Коучи: анализируйте свои сессии
- Находите слепые пятна
- Совершенствуйте подход

### Пример исследовательского вопроса

**Вопрос:** "Какие техники наиболее эффективны для людей с комбинацией перфекционизм + страх оценки?"

**Метод:**
1. Соберите 15+ логов с этой комбинацией паттернов
2. Проанализируйте раздел "Techniques Used" и "User Reaction"
3. Посмотрите на "Outcomes" — кто действительно начал действовать
4. Выявите корреляции между техниками и успехом
5. Синтезируйте находки в рекомендации

**Результат:** "Для перфекционизм + страх оценки наиболее эффективны: time-boxing (7/8 позитивных реакций), permission to be imperfect (6/8), low-stakes starting (8/8). Менее эффективны: публичные обязательства (усиливают страх)."

## Примеры использования

### Пример 1: Студент с дипломной работой
- **Паттерн:** Overwhelm + Fear of failure
- **Техники:** Extreme decomposition, Two-minute rule, Self-compassion
- **Результат:** План с микрошагами по 15 минут
- **Лог:** Полный — для отслеживания прогресса через месяц

### Пример 2: Профессионал с self-review
- **Паттерн:** Perfectionism + Imposter syndrome
- **Техники:** Evidence gathering, "Done > Perfect", Self-as-friend
- **Результат:** Конкретный план на 4 дня
- **Лог:** Полный — богатый исследовательский кейс (см. example)

### Пример 3: Фрилансер с новым проектом
- **Паттерн:** Fear of evaluation + Lack of motivation
- **Техники:** Value connection, Decatastrophizing, Fresh start ritual
- **Результат:** Связь с ценностями + первый шаг
- **Лог:** Быстрый — короткая консультация

## Частые вопросы

### Q: Будет ли Claude автоматически создавать логи?
**A:** Нет, только по запросу пользователя. После завершения сессии Claude предложит создать лог, но вы можете отказаться.

### Q: Насколько детальным должен быть лог?
**A:** Зависит от цели. Для личного использования — любой уровень детальности. Для исследований — чем детальнее, тем лучше.

### Q: Можно ли редактировать логи после создания?
**A:** Да! Это markdown-файлы, можете дополнять заметками, результатами, обновлениями прогресса.

### Q: Что если я не хочу делиться данными?
**A:** Совершенно нормально! Логи — это в первую очередь ваш инструмент. Используйте для себя, делитесь только если комфортно.

### Q: Как обеспечить анонимность?
**A:** Следуйте guidelines в секции Privacy. Заменяйте имена, компании, специфичные детали. Сохраняйте паттерны и инсайты, но обобщайте контекст.

### Q: Сколько логов нужно для исследования?
**A:** Минимум 10-15 для базовых паттернов. 20-30+ для серьёзного анализа. Зависит от вопроса исследования.
