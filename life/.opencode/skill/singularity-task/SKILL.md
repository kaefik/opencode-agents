# Управление задачами Singularity

## Обзор

Этот навык обеспечивает управление задачами, организацию проектов, отслеживание привычек и создание записных книжек с помощью инструментов Singularity MCP. Он предоставляет всеобъемлющие рекомендации по работе с часовыми поясами, проектами, задачами, записными книжками и привычками.

## Проекты пользователя

Когда пользователь просит добавить задачи или упоминает какую-либо деятельность, обратитесь к файлу `.opencode/skill/singularity-task/references/user_projects.md`, чтобы определить соответствующий проект. Этот справочник содержит личный список проектов пользователя с описаниями и рекомендациями по использованию.

## Конфигурация часового пояса

**КРИТИЧНО ВАЖНО:** Все даты и время указаны в часовом поясе GMT+3, если пользователь не указал иное.

- Часовой пояс по умолчанию: GMT+3
- Когда в разговоре указано время, интерпретируйте его как GMT+3
- Замените GMT+3 на фактический часовой пояс пользователя, если это необходимо

## Работа с проектами

### Создание проектов

При создании проектов следуйте этим требованиям:

**Формат эмодзи для проекта:**
- Должен быть шестнадцатеричным кодом Unicode **без префикса**
- Формат: `"1f49e"` (только шестнадцатеричная строка)
- НЕ формат Unicode: `"U+1F49E"`
- НЕ сам эмодзи: `"💞"`

Примеры:
- 💞 = `"1f49e"`
- 🎯 = `"1f3af"`
- 🚀 = `"1f680"`
- ⭐️ = `"2b50"`

### Добавление задач в проекты

**Поведение по умолчанию:** Всегда помещайте новые задачи в базовую группу задач (base task group) проекта, если иное не указано явно.

### Классификация задач к проектам

**Классификация:** При упоминании задачи или запроса, определите подходящий проект.

**Процесс:**
1. Проанализируйте контекст запроса и определите подходящий проект
2. Используйте `.opencode/skill/singularity-task/references/user_projects.md` как справочник для классификации задач
3. Если возникают затруднения при определении проекта — спросите пользователя с вариантами подходящих проектов по смыслу задачи
4. Если пользователь не выбрал ни один вариант — отнесите задачу в базовую группу задач (base task group) проекта

**Приоритет проектов:**
- Сначала определите наиболее подходящий проект
- Затем используйте вторичный анализ для уточнения выбора
- Учитывайте контекст и цели пользователя

## Настройки проекта

**Поле `showInBasket`:** Не устанавливайте это поле, если пользователь специально не попросил об этом.

### Заметки к проекту

**Формат содержимого заметки:** Должен быть в формате Delta
- Передавайте массив операций напрямую: `[{},...]`
- НЕ упакованный формат: `{"ops":[{},...]}`
- **Важно:** Последняя вставка (insert) в Delta всегда должна заканчиваться переносом строки

Пример:
```json
[
  {"insert": "Первая строка\n"},
  {"insert": "Вторая строка\n"}
]
```

## Работа с задачами

### Формат вывода задач

При выводе списка задач следуйте этому формату:

**Структура вывода:**
1. **Первая строка** — название задачи **жирным шрифтом** (markdown: `**текст**`)
2. **Вторая строка** — проект задачи, дата начала работы и дата конца работы (если есть)
3. **Третья и последующие строки** — информация о запланированном времени и напоминании, приоритет

**Сортировка:**
- Сначала задачи по дате начала (от самых ранних к поздним)
- Задачи без даты — в конце списка

**Пример формата:**

```
**do флюрографию**

Проект: Тело и Здоровье
Дата начала: 2026-06-30T21:00:00.000Z
Дата окончания: (нет)
Запланирована на конкретное время: Да (21:00)
Установлено напоминание: Да
Приоритет: 1

**сдать анализ дыхательный на хеликобактер**

Проект: Тело и Здоровье
Дата начала: 2026-03-11T21:00:00.000Z
Дата окончания: (нет)
Запланирована на конкретное время: Да (21:00)
Установлено напоминание: Да
Приоритет: 1
```

**Форматирование даты и времени:**
- Используйте формат ISO: `YYYY-MM-DDTHH:mm:ss.sssZ`
- Для задач без времени используйте дату без времени
- Если даты начала и окончания совпадают, укажите только одну

### Вывод задач из проекта

**Правила вывода задач из конкретного проекта:**
1. Получите список всех задач из указанного проекта
2. Отсортируйте их по полю `start` (от самых ранних к поздним)
3. Выведите в соответствии с форматом, описанным выше
4. Для повторяющихся задач (recurrence) укажите тип повторения и следующую дату выполнения

**Пример с реальными данными (проект "Тело и Здоровье"):**

```
**начать подготовку к дыхательному тесту**

Проект: Тело и Здоровье
Дата начала: 2026-02-28T21:00:00.000Z
Дата окончания: (нет)
Запланирована на конкретное время: Да (21:00)
Установлено напоминание: Да
Приоритет: 1

**Do УЗИ брюшной полости**

Проект: Тело и Здоровье
Дата начала: 2026-02-15T21:00:00.000Z
Дата окончания: (нет)
Запланирована на конкретное время: Да (21:00)
Установлено напоминание: Да
Приоритет: 1

**Do упражнения на пресс и ноги**

Проект: Тело и Здоровье
Дата начала: 2026-04-16T18:45:00.000Z
Дата окончания: (нет)
Запланирована на конкретное время: Да (18:45)
Установлено напоминание: Да
Приоритет: 1

**установка импланта**

Проект: Тело и Здоровье
Дата начала: (нет)
Дата окончания: (нет)
Запланирована на конкретное время: Нет
Установлено напоминание: Да
Приоритет: 1
```

### Обработка даты и времени

**Часовой пояс:** Используйте GMT+3 для всех дат и времени задач.

**Установка даты и времени:**
- Используйте поле `start` для указания даты/времени начала задачи
- Используйте поле `deadline` для указания дедлайна (даты окончания)
- Для задач с точным временем используйте `useTime: true`
- Для задач только с датой используйте `useTime: false`

**Правила:**
- `start` — когда задача становится активной (необходимо указать для новых задач)
- `deadline` — крайний срок завершения (необязательно)
- `useTime: false` — дата без конкретного времени
- `useTime: true` — дата с указанным временем

### Уведомления

**Формат:**
```json
{
  "notifies": [60, 15],      // Массив в минутах, от большего к меньшему
  "notify": 1,               // Основное уведомление включено
  "alarmNotify": true        // Будильник (по умолчанию false)
}
```

**Правила:**
- Массив `notifies`: Минуты до события, упорядоченные от большего к меньшему
- Пример: `[60, 15]` = уведомления за 1 час и за 15 минут до
- `notify: 1` включает основное уведомление
- `alarmNotify: true` только при явном запросе (по умолчанию: false)

### Приоритет задачи

**Приоритет по умолчанию:** `1`

### Интерпретация дат

**Запрос даты при создании задачи:**
- Если пользователь не указывает дату для задачи, спросите пользователя: "На какую дату создать задачу? Выберите: на сегодня, без даты или укажите другую дату"
- `start: "2026-02-08"` — дата без указания времени (useTime: false)
- `start: "2026-02-08T18:00"` — дата с указанием времени (useTime: true)

**Технические поля:**
- `modifiedDate` — Временная метка последнего изменения (НЕ время завершения)
- `createdDate` — Исходная временная метка создания
- Большая разница между `modifiedDate` и `createdDate` указывает на то, что задача была перенесена

## Записные книжки и заметкоподобные записи

### Создание записных книжек

**Когда пользователь просит записную книжку:** Создайте проект с атрибутом `notebook`, установленным в `true`.

### Добавление элементов в записные книжки

**Поведение по умолчанию:** Создавайте задачу с атрибутом `isNote` при добавлении элементов в записную книжку.

### Задачи против Заметок

**Дерево решений:**
1. Пользователь явно просит задачу → Создать задачу (без `isNote`)
2. Пользователь добавляет элемент в записную книжку → Создать задачу с атрибутом `isNote`
3. Пользователь хочет заметку в проекте, не являющемся записной книжкой → Создать задачу с атрибутом `isNote`

## Работа с привычками

### Цвета привычек

**Формат:** Передавайте цвет в виде строки из списка допустимых значений, а НЕ цветовой код.

**Допустимые значения:**
- `"red"` (красный)
- `"pink"` (розовый)
- `"purple"` (фиолетовый)
- `"deepPurple"` (темно-фиолетовый)
- `"indigo"` (индиго)
- `"lightBlue"` (голубой)
- `"cyan"` (циан)
- `"teal"` (бирюзовый)
- `"green"` (зеленый)
- `"lightGreen"` (светло-зеленый)
- `"lime"` (лаймовый)
- `"yellow"` (желтый)
- `"amber"` (янтарный)
- `"orange"` (оранжевый)
- `"deepOrange"` (темно-оранжевый)
- `"brown"` (коричневый)
- `"grey"` (серый)
- `"blueGrey"` (серо-голубой)

### Статус привычки

**Активные привычки:** Статус `0`

**Всегда создавайте привычки со статусом:** `status: 0`

## Типичные рабочие процессы


### Пример: Создание задачами

```
Пользователь: "Создай задачу покупки продуктов на завтра в 18:00"

Шаги:
1. Понять к какому проекту относится задача (например, "Купить домой") , используя `.opencode/skill/singularity-task/references/user_projects.md` 
2. Создать задачу с именем "Покупка продуктов" с указанием проекта "Купить домой"
3. Установить `start: "2026-02-08T18:00"` и `useTime: true` для GMT+3
4. Установить приоритет 1 (по умолчанию)
5. Использовать GMT+3 для любых указаний даты/времени
```

### Пример: Создание проекта с задачами

```
Пользователь: "Создай проект 'Редизайн сайта' с задачами на дизайн, разработку и тестирование"

Шаги:
1. Создать проект с именем "Редизайн сайта"
2. Установить подходящий эмодзи (например, "1f4bb" для 💻)
3. Добавить задачи в базовую группу задач:
   - Задача на дизайн
   - Задача на разработку
   - Задача на тестирование
4. Установить приоритет 1 для всех задач (по умолчанию)
5. Для задач с датой установить start: "2026-02-08"
6. Использовать GMT+3 для любых указаний даты/времени
```

### Пример: Создание записной книжки

```
Пользователь: "Создай исследовательскую записную книжку для моего ИИ-проекта"

Шаги:
1. Создать проект с именем "AI Research" (или "Исследования ИИ")
2. Установить notebook: true
3. Установить подходящий эмодзи (например, "1f4d3" для 📓)
4. При добавлении записей создавать задачи с isNote: true
```

### Пример: Создание привычки

```
Пользователь: "Создай ежедневную привычку чтения"

Шаги:
1. Создать привычку с подходящим именем
2. Установить цвет из допустимого списка (например, "blue")
3. Установить status: 0 (активная)
```
