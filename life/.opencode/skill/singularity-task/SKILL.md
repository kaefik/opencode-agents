# Управление задачами Singularity

## Обзор

Этот навык обеспечивает управление задачами, организацию проектов, отслеживание привычек и создание записных книжек с помощью инструментов Singularity MCP. Он предоставляет всеобъемлющие рекомендации по работе с часовыми поясами, проектами, задачами, записными книжками и привычками.

## Проекты пользователя

Когда пользователь просит добавить задачи или упоминает какую-либо деятельность, обратитесь к файлу `references/user_projects.md`, чтобы определить соответствующий проект. Этот справочник содержит личный список проектов пользователя с описаниями и рекомендациями по использованию.

## Конфигурация часового пояса

**КРИТИЧНО ВАЖНО:** Singularity API работает с часовыми поясами. Пользователь находится в GMT+3 (Казань).

### Формат хранения в Singularity

- **API хранит все даты в формате UTC** (ISO 8601 с суффиксом `Z`)
- Пример: `2026-02-10T21:00:00.000Z` — это время в UTC

### Работа с пользователем

- **Ввод пользователя:** интерпретируйте как локальное время GMT+3
- **Отправка в API:** конвертируйте локальное время в UTC
- **Вывод пользователю:** конвертируйте UTC в локальное время GMT+3

### Примеры конвертации

| Локальное время (GMT+3) | UTC (для API) |
|------------------------|---------------|
| 2026-02-11 00:00 | 2026-02-10T21:00:00.000Z |
| 2026-02-11 12:00 | 2026-02-11T09:00:00.000Z |
| 2026-02-11 18:30 | 2026-02-11T15:30:00.000Z |

### Правила конвертации

**При создании/обновлении задач:**
- Пользователь говорит: "создай задачу на завтра в 18:00" → интерпретировать как 2026-02-12 18:00 GMT+3
- Отправить в API: `start: "2026-02-12T15:00:00.000Z"` (вычесть 3 часа)

**При выводе задач:**
- API возвращает: `start: "2026-02-10T21:00:00.000Z"`
- Показать пользователю: "Дата начала: 2026-02-11 00:00" (прибавить 3 часа)

## Работа с проектами

### Создание проектов

При создании проектов следуйте этим требованиям:

**Формат эмодзи для проекта:**
- Должен быть шестнадцатеричным кодом Unicode **без префикса**
- Формат: `"1f49e"` (только шестнадцатеричная строка)
- НЕ формат Unicode: `"U+1F49E"`
- НЕ сам эмодзи: `"💞"`

Примеры:
- 💞 = `"1f49e"`
- 🎯 = `"1f3af"`
- 🚀 = `"1f680"`
- ⭐️ = `"2b50"`

### Добавление задач в проекты

**Поведение по умолчанию:** Всегда помещайте новые задачи в базовую группу задач (base task group) проекта, если иное не указано явно.

### Классификация задач к проектам

**Классификация:** При упоминании задачи или запроса, определите подходящий проект.

**Процесс:**
1. Проанализируйте контекст запроса и определите подходящий проект
2. Используйте `references/user_projects.md` как справочник для классификации задач
3. Если возникают затруднения при определении проекта — спросите пользователя с вариантами подходящих проектов по смыслу задачи
4. Если пользователь не выбрал ни один вариант — отнесите задачу в базовую группу задач (base task group) проекта

**Приоритет проектов:**
- Сначала определите наиболее подходящий проект
- Затем используйте вторичный анализ для уточнения выбора
- Учитывайте контекст и цели пользователя

## Настройки проекта

**Поле `showInBasket`:** Не устанавливайте это поле, если пользователь специально не попросил об этом.

### Заметки к проекту

**Формат содержимого заметки:** Должен быть в формате Delta
- Передавайте массив операций напрямую: `[{},...]`
- НЕ упакованный формат: `{"ops":[{},...]}`
- **Важно:** Последняя вставка (insert) в Delta всегда должна заканчиваться переносом строки

Пример:
```json
[
  {"insert": "Первая строка\n"},
  {"insert": "Вторая строка\n"}
]
```

## Работа с задачами

### Формат вывода задач

При выводе списка задач следуйте этому формату:

**Структура вывода:**
1. **Первая строка** — название задачи **жирным шрифтом** (markdown: `**текст**`)
2. **Вторая строка** — проект задачи, дата начала работы и дата конца работы (если есть)
3. **Третья и последующие строки** — информация о запланированном времени и напоминании, приоритет

**Сортировка:**
- Сначала задачи по дате начала (от самых ранних к поздним)
- Задачи без даты — в конце списка

**Пример формата:**

```
**do флюрографию**

Проект: Тело и Здоровье
Дата начала: 2026-07-01 00:00
Дата окончания: (нет)
Запланирована на конкретное время: Да (00:00)
Установлено напоминание: Да
Приоритет: 1

**сдать анализ дыхательный на хеликобактер**

Проект: Тело и Здоровье
Дата начала: 2026-03-12 00:00
Дата окончания: (нет)
Запланирована на конкретное время: Да (00:00)
Установлено напоминание: Да
Приоритет: 1
```

**Форматирование даты и времени:**
- Выводите даты в локальном времени GMT+3 (конвертируйте из UTC)
- Формат для вывода: `YYYY-MM-DD HH:mm` (локальное время)
- Для задач без времени: `YYYY-MM-DD` (дата начала дня в GMT+3)
- Пример конвертации: `2026-02-10T21:00:00.000Z` → `2026-02-11 00:00`
- Если даты начала и окончания совпадают, укажите только одну

### Вывод задач из проекта

**Правила вывода задач из конкретного проекта:**
1. Получите список всех задач из указанного проекта
2. Отсортируйте их по полю `start` (от самых ранних к поздним)
3. Выведите в соответствии с форматом, описанным выше
4. Для повторяющихся задач (recurrence) укажите тип повторения и следующую дату выполнения

**Пример с реальными данными (проект "Тело и Здоровье"):**

```
**начать подготовку к дыхательному тесту**

Проект: Тело и Здоровье
Дата начала: 2026-03-01 00:00
Дата окончания: (нет)
Запланирована на конкретное время: Да (00:00)
Установлено напоминание: Да
Приоритет:1

**Do УЗИ брюшной полости**

Проект: Тело и Здоровье
Дата начала: 2026-02-16 00:00
Дата окончания: (нет)
Запланирована на конкретное время: Да (00:00)
Установлено напоминание: Да
Приоритет:1

**Do упражнения на пресс и ноги**

Проект: Тело и Здоровье
Дата начала: 2026-04-16 21:45
Дата окончания: (нет)
Запланирована на конкретное время: Да (21:45)
Установлено напоминание: Да
Приоритет:1

**установка импланта**

Проект: Тело и Здоровье
Дата начала: (нет)
Дата окончания: (нет)
Запланирована на конкретное время: Нет
Установлено напоминание: Да
Приоритет:1
```

### Обработка даты и времени

**Часовой пояс:**
- Локальное время пользователя: GMT+3
- API хранит даты в формате UTC
- При отправке в API: конвертируйте GMT+3 → UTC (вычесть 3 часа)
- При выводе пользователю: конвертируйте UTC → GMT+3 (прибавить 3 часа)

**Установка даты и времени при создании задач:**

Формат даты в API всегда должен быть ISO 8601 с суффиксом `Z` (UTC):
```
start: "2026-02-11T15:00:00.000Z"  // UTC формат
```

**Конвертация при создании задачи:**
- Пользователь: "создай задачу на 2026-02-11 в 18:00" (локальное GMT+3)
- Конвертация: 2026-02-11 18:00 GMT+3 → 2026-02-11 15:00 UTC
- Отправить в API: `start: "2026-02-11T15:00:00.000Z"`

**Использование полей:**
- `start` — когда задача становится активной (необходимо указать для новых задач), в формате UTC
- `deadline` — крайний срок завершения (необязательно), в формате UTC
- `useTime: false` — дата без конкретного времени (время задачи не имеет значения)
- `useTime: true` — задача запланирована на конкретное время

**Правила конвертации даты без времени:**
- Пользователь: "создай задачу на 11 февраля"
- Локальная дата: 2026-02-11 (все время дня, GMT+3)
- Конвертация: начало дня в GMT+3 = 2026-02-11 00:00 → 2026-02-10 21:00 UTC
- Отправить в API: `start: "2026-02-10T21:00:00.000Z"`, `useTime: false`

**Правила конвертации даты с временем:**
- Пользователь: "создай задачу на 11 февраля в 18:00"
- Локальная дата/время: 2026-02-11 18:00 (GMT+3)
- Конвертация: 2026-02-11 18:00 GMT+3 → 2026-02-11 15:00 UTC
- Отправить в API: `start: "2026-02-11T15:00:00.000Z"`, `useTime: true`

### Уведомления

**Формат:**
```json
{
  "notifies": [60, 15],      // Массив в минутах, от большего к меньшему
  "notify": 1,               // Основное уведомление включено
  "alarmNotify": true        // Будильник (по умолчанию false)
}
```

**Правила:**
- Массив `notifies`: Минуты до события, упорядоченные от большего к меньшему
- Пример: `[60, 15]` = уведомления за 1 час и за 15 минут до
- `notify: 1` включает основное уведомление
- `alarmNotify: true` только при явном запросе (по умолчанию: false)

### Приоритет задачи

**Приоритет по умолчанию:** `1`

### Интерпретация дат

**Запрос даты при создании задачи:**
- Если пользователь не указывает дату для задачи, спросите пользователя: "На какую дату создать задачу? Выберите: на сегодня, без даты или укажите другую дату"

**Примеры конвертации при отправке в API:**

Дата без времени (задача на весь день):
```
Пользователь: "создай задачу на 8 февраля"
Локально (GMT+3): 2026-02-08 00:00
Отправить в API: start: "2026-02-07T21:00:00.000Z", useTime: false
```

Дата с временем:
```
Пользователь: "создай задачу на 8 февраля в 18:00"
Локально (GMT+3): 2026-02-08 18:00
Отправить в API: start: "2026-02-08T15:00:00.000Z", useTime: true
```

**Технические поля:**
- `modifiedDate` — Временная метка последнего изменения (НЕ время завершения), в формате UTC
- `createdDate` — Исходная временная метка создания, в формате UTC
- Большая разница между `modifiedDate` и `createdDate` указывает на то, что задача была перенесена

**Правила фильтрации "задач на сегодня":**

**КРИТИЧНО ВАЖНО:** При запросе пользователя "покажи задачи на сегодня" или "какие задачи на сегодня", фильтруйте задачи ТОЛЬКО по полю `start` (дата начала задачи).

**НЕ включайте в список:**
- Задачи с `createdDate` = сегодня, но `start` ≠ сегодня (это задачи, созданные сегодня, но запланированные на другую дату)
- Задачи с `journalDate` = сегодня, но `start` ≠ сегодня

**Включайте в список:**
- Только задачи, у которых поле `start` попадает в диапазон сегодняшнего дня (00:00-23:59 локального времени GMT+3)

**Пример:**

| Название | start (локально) | createdDate (локально) | Включать? |
|----------|-------------------|------------------------|-----------|
| do budget | 2026-02-12 18:00 | 2026-02-11 16:00 | ❌ (запланирована на завтра) |
| Buy groceries | 2026-02-11 12:00 | 2026-02-10 15:00 | ✅ (запланирована на сегодня) |
| Report | 2026-02-11 00:00 | 2026-02-11 09:00 | ✅ (запланирована на сегодня) |

## Записные книжки и заметкоподобные записи

### Создание записных книжек

**Когда пользователь просит записную книжку:** Создайте проект с атрибутом `notebook`, установленным в `true`.

### Добавление элементов в записные книжки

**Поведение по умолчанию:** Создавайте задачу с атрибутом `isNote` при добавлении элементов в записную книжку.

### Задачи против Заметок

**Дерево решений:**
1. Пользователь явно просит задачу → Создать задачу (без `isNote`)
2. Пользователь добавляет элемент в записную книжку → Создать задачу с атрибутом `isNote`
3. Пользователь хочет заметку в проекте, не являющемся записной книжкой → Создать задачу с атрибутом `isNote`

## Работа с привычками

### Цвета привычек

**Формат:** Передавайте цвет в виде строки из списка допустимых значений, а НЕ цветовой код.

**Допустимые значения:**
- `"red"` (красный)
- `"pink"` (розовый)
- `"purple"` (фиолетовый)
- `"deepPurple"` (темно-фиолетовый)
- `"indigo"` (индиго)
- `"lightBlue"` (голубой)
- `"cyan"` (циан)
- `"teal"` (бирюзовый)
- `"green"` (зеленый)
- `"lightGreen"` (светло-зеленый)
- `"lime"` (лаймовый)
- `"yellow"` (желтый)
- `"amber"` (янтарный)
- `"orange"` (оранжевый)
- `"deepOrange"` (темно-оранжевый)
- `"brown"` (коричневый)
- `"grey"` (серый)
- `"blueGrey"` (серо-голубой)

### Статус привычки

**Активные привычки:** Статус `0`

**Всегда создавайте привычки со статусом:** `status: 0`

## Типичные рабочие процессы


### Пример: Создание задачами

```
Пользователь: "Создай задачу покупки продуктов на завтра в 18:00"

Шаги:
1. Понять к какому проекту относится задача (например, "Купить домой"), используя `references/user_projects.md`
2. Создать задачу с именем "Покупка продуктов" с указанием проекта "Купить домой"
3. Конвертировать локальное время в UTC:
   - Локально (GMT+3): 2026-02-08 18:00
   - UTC: 2026-02-08 15:00
4. Установить `start: "2026-02-08T15:00:00.000Z"` и `useTime: true`
5. Установить приоритет 1 (по умолчанию)
```

### Пример: Создание проекта с задачами

```
Пользователь: "Создай проект 'Редизайн сайта' с задачами на дизайн, разработку и тестирование"

Шаги:
1. Создать проект с именем "Редизайн сайта"
2. Установить подходящий эмодзи (например, "1f4bb" для 💻)
3. Добавить задачи в базовую группу задач:
    - Задача на дизайн
    - Задача на разработку
    - Задача на тестирование
4. Установить приоритет 1 для всех задач (по умолчанию)
5. Для задач с датой выполнить конвертацию GMT+3 → UTC:
   - Локально: 2026-02-08 00:00
   - Отправить: start: "2026-02-07T21:00:00.000Z"
```

### Пример: Создание записной книжки

```
Пользователь: "Создай исследовательскую записную книжку для моего ИИ-проекта"

Шаги:
1. Создать проект с именем "AI Research" (или "Исследования ИИ")
2. Установить notebook: true
3. Установить подходящий эмодзи (например, "1f4d3" для 📓)
4. При добавлении записей создавать задачи с isNote: true
```

### Пример: Создание привычки

```
Пользователь: "Создай ежедневную привычку чтения"

Шаги:
1. Создать привычку с подходящим именем
2. Установить цвет из допустимого списка (например, "blue")
3. Установить status: 0 (активная)
```
