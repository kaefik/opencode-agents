## Подробная структура сущностей

### Проекты (`projects`)

Поля:
- `id` - Уникальный идентификатор (формат: `P-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`)
- `title` - Название проекта
- `start` - Дата начала (ISO 8601)
- `end` - Дата окончания (ISO 8601)
- `note` - ID заметки проекта
- `tags` - Массив ID тегов
- `parent` - ID родительского проекта
- `emoji` - Эмодзи проекта
- `color` - Цвет проекта (HEX)
- `showInBasket` - Показывать в корзине
- `isNotebook` - Является записной книжкой
- `journalDate` - Дата журнала
- `_removed` - Флаг удаления

Пример:
```json
{
  "id": "P-195bba6f-6dd3-4cfe-914e-c283a525e694",
  "title": "Agile Life",
  "start": "",
  "end": "",
  "parent": "N/A",
  "emoji": "",
  "color": "#c62828",
  "tags": [],
  "note": ""
}
```

### Задачи (`tasks`)

Поля:
- `id` - Уникальный идентификатор (формат: `T-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`)
- `title` - Название задачи
- `note` - ID заметки задачи
- `projectId` - ID проекта
- `group` - ID группы задач
- `parent` - ID родительской задачи (для подзадач)
- `start` - Дата начала (ISO 8601)
- `deadline` - Дедлайн (ISO 8601)
- `createdDate` - Дата создания (ISO 8601)
- `journalDate` - Дата журнала (ISO 8601)
- `timeLength` - Длительность в минутах
- `priority` - Приоритет (0 = низкий, 1 = высокий, 2 = срочный)
- `complete` - Процент выполнения (0-100)
- `state` - Состояние
- `checked` - Отмечено (0 = не выполнена, 1 = выполнена)
- `showInBasket` - Показывать в корзине (True = удалена)
- `deleteDate` - Дата удаления (ISO 8601)
- `deferred` - Отложено
- `useTime` - Использовать время
- `tags` - Массив ID тегов
- `notifies` - Массив минут до напоминания (например: `[60, 30]`)
- `startNotifiesReaded` - Прочитанные напоминания старта
- `alarmNotify` - Будильник
- `deadlineNotifyReaded` - Прочитанные напоминания дедлайна
- `recurrence` - Настройки повторения
- `scheduleOrder` - Порядок в расписании
- `parentOrder` - Порядок среди подзадач
- `seenToday` - Дата просмотра сегодня
- `isNote` - Является заметкой
- `progress` - Прогресс
- `_removed` - Флаг удаления

Пример:
```json
{
  "id": "T-003ea796-858f-4155-b657-20070a59e560",
  "title": "do программу котора будет создавать по массиву людей графиеское прдестваление семейного древа",
  "note": "N-T-003ea796-858f-4155-b657-20070a59e560",
  "projectId": "P-e607e25e-0f0c-4034-b3bc-b85c928e5b1d",
  "createdDate": "2025-10-07T14:40:05.473Z",
  "priority": 1,
  "complete": 0,
  "state": 1,
  "checked": 0,
  "tags": []
}
```

### Повторение задач (`recurrence`)

Поля внутри `recurrence`:
- `ending` - Настройки окончания повторения
  - `type` - Тип окончания (`NONE`, `DATE`, `ITERATIONS`, `LIMIT`)
  - `iterations` - Количество итераций
  - `limit` - Лимит
  - `date` - Дата окончания
- `repeat` - Настройки повторения
  - `type` - Тип (`EVERYDAY`, `EVERYWEEK`, `EVERYMONTH`, `EVERYYEAR`)
  - `days` - Дни недели (для `EVERYWEEK`)
  - `interval` - Интервал
- `paused` - Пауза
- `time` - Время
- `lastGeneratedTaskId` - ID последней созданной задачи
- `lastTime` - Последнее время
- `nextTime` - Следующее время
- `startTime` - Время старта
- `onlyOnComplete` - Только при завершении
- `startAfterComplete` - Начать после завершения
- `alarmNotify` - Уведомление будильником
- `timeLength` - Длительность

Пример:
```json
{
  "recurrence": {
    "ending": {
      "type": "NONE",
      "iterations": 72,
      "limit": 5
    },
    "repeat": {
      "type": "EVERYWEEK",
      "days": [6],
      "interval": 1
    },
    "paused": false,
    "nextTime": "2027-03-12T21:00:00.000Z",
    "startTime": "2025-10-24T21:00:00.000Z"
  }
}
```

### Группы задач (`taskGroups`)

Поля:
- `id` - Уникальный идентификатор (формат: `Q-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`)
- `title` - Название группы
- `parent` - ID родительского элемента (обычно проект)
- `parentOrder` - Порядок в родителе
- `fake` - Фейковая группа

Пример:
```json
{
  "id": "Q-05f740d6-dc21-44fe-a3bf-7d568198c04a",
  "title": "🎅 Подготовка подарков",
  "parent": "P-bf80800c-44d8-43fa-ac54-92389a280fa6",
  "parentOrder": 900,
  "fake": false
}
```

### Заметки (`notes`)

Поля:
- `id` - Уникальный идентификатор (формат: `N-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`)
- `containerId` - ID контейнера (задачи или проекта)
- `content` - Содержимое в формате Quill Delta JSON

Пример:
```json
{
  "id": "N-T-003ea796-858f-4155-b657-20070a59e560",
  "containerId": "T-003ea796-858f-4155-b657-20070a59e560",

}
```

## Статус выполнения задач

В Singularity есть два поля для отслеживания статуса выполнения:

### `checked` - флаг выполнения задачи
- `checked = 0`: задача не выполнена
- `checked = 1`: задача выполнена

Это основное поле, определяющее статус выполнения задачи.

### `complete` - процент выполнения (для задач с подзадачами)
- Значения от 0 до 100
- Используется для родительских задач, у которых есть подзадачи
- Показывает процент выполненных подзадач

**Примечание:** В данном бэкапе все задачи имеют `complete = 0`, так как только 30 задач имеют подзадачи и ни одна из них не имеет частичного прогресса.

## Удаленные задачи (Корзина)

Задачи могут быть удалены в корзину. Состояние задачи определяется двумя полями:

### `showInBasket` - флаг нахождения в корзине
- `showInBasket = True`: задача удалена и находится в корзине
- `showInBasket = False` или отсутствует: задача активна

### `deleteDate` - дата удаления
- Содержит дату и время удаления задачи в формате ISO 8601
- Присутствует у всех задач в корзине
- Пример: `"2026-02-07T20:04:24.780Z"`

### Статистика корзины

| Категория | Количество | Процент |
|-----------|-----------|----------|
| Задачи в корзине (`showInBasket=True`) | 158 | 5.0% |
| Из них выполненные (`checked=1`) | 7 | 4.4% |
| Из них не выполненные (`checked=0`) | 151 | 95.6% |

### Пример удаленной задачи

```json
{
  "id": "T-c24746c5-f0e3-4758-b4e8-4d09144be24e",
  "title": "Установить операционную систему на старый компьютер то есть ноутбук",
  "checked": 0,
  "showInBasket": true,
  "deleteDate": "2026-02-07T20:04:24.780Z",
  "deferred": true
}
```

Эта задача:
- Не выполнена (`checked = 0`)
- Находится в корзине (`showInBasket = true`)
- Была удалена 2026-02-07 в 20:04
- Была отложена (`deferred = true`)

## Связи между сущностями

1. **Задача → Проект**: через `projectId`
2. **Задача → Группа**: через `group`
3. **Задача → Родительская задача**: через `parent`
4. **Задача → Заметка**: через `note`
5. **Задача → Теги**: через массив `tags`
6. **Задача → Корзина**: через `showInBasket` (True = в корзине)
7. **Задача → Дата удаления**: через `deleteDate`
8. **Проект → Родительский проект**: через `parent`
9. **Проект → Заметка**: через `note`
10. **Проект → Теги**: через массив `tags`
11. **Группа → Проект**: через `parent`
12. **Заметка → Контейнер**: через `containerId` (ID задачи или проекта)
